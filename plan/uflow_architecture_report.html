UFlow 服务侧架构与重构报告

更新日期：2026-06-22 · 范围：daemon / SDK / monitor / trace / ACL backend · 不包含 pypto-serving 适配

1. 结论

接口收敛

UFlow public API 统一到 DataObject + Placement + TransferPlan + TransferEvent。旧 buffer alias 和 client-local transfer helper 不再作为正式接口。

模块边界

daemon 已按 router、object、planner、executor、backend、stats、trace、capabilities 分层，ACL backend 进一步拆成 runtime / HBM / host / copy。

验收方式

本轮不以 pypto-serving 为验收目标，先用裸 UFlow 覆盖 HBM、DDR、HBM-DDR、HBM-HBM、DDR-DDR、monitor 和 trace。

架构审视结果：当前方向合理，但原型阶段留下的旧 public op、SDK 中 client-local copy helper、backend 文件职责混杂会阻碍后续扩展。本轮重构将这些路径移出正式边界。

7. Monitor 与 Trace

Monitor

常开低频聚合

/healthz、/v1/stats、/v1/objects、/v1/capabilities、/v1/observability 用于 UI/CLI 的服务状态监控。

Trace

默认关闭事件级Chrome trace

/v1/trace/start|stop|flush 控制细粒度采集；正式产物为 trace_events.json、trace_summary.json、trace_summary.csv、summary.md。
